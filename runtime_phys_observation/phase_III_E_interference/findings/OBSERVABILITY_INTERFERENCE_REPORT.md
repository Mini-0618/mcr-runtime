## Key Finding: WAL Hash Divergence ≠ Physics Divergence

The initial test showed "WAL hash mismatch" (different ordering hashes). Investigation revealed:

- **Root cause**: `memory_id` generated by `uuid.uuid4()` — NOT seeded by `random.seed()`
- **Both runs produce**: identical seq, tick, event type, from_state, to_state
- **Functional event tuples match**: True (19971/19971 entries)
- **Lifecycle hash (event_type sequence)**: MATCH ✅

**Conclusion**: WAL hash divergence is expected Python behavior (uuid4 non-determinism).
The observer does **not** alter the event sequence or lifecycle.
